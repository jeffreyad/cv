# CV Repository Structure

```
cv-repo/
├── .github/
│   └── workflows/
│       └── render-cv.yml          # GitHub Actions workflow for auto-rendering
│
├── assets/
│   ├── README.md                  # Instructions for custom Word template
│   └── styles.css                 # Custom CSS styling for HTML output
│
├── archive/
│   └── README.md                  # Instructions for archiving old versions
│
├── docs/                          # Generated by Quarto (GitHub Pages source)
│   ├── cv.html                    # HTML version (auto-generated)
│   ├── cv.pdf                     # PDF version (auto-generated)
│   └── cv.docx                    # Word version (auto-generated)
│
├── .gitignore                     # Git ignore file
├── _quarto.yml                    # Quarto project configuration
├── cv.qmd                         # Main CV source file (EDIT THIS!)
├── LICENSE                        # MIT License
├── README.md                      # Repository overview and links
└── SETUP.md                       # Setup and deployment instructions

```

## Key Files to Edit

### Primary Files (Edit These)
- **cv.qmd** - Your CV content (main file to update)
- **README.md** - Update links and contact info
- **_quarto.yml** - Update GitHub/LinkedIn URLs

### Styling (Optional)
- **assets/styles.css** - Customize colors, fonts, layout

### Configuration (Usually Don't Need to Change)
- **.github/workflows/render-cv.yml** - Auto-render workflow
- **_quarto.yml** - Quarto settings
- **.gitignore** - Files to exclude from git

## Generated Files (Don't Edit Directly)

The `docs/` folder is automatically generated when you run `quarto render`:
- `docs/cv.html` - HTML version
- `docs/cv.pdf` - PDF version
- `docs/cv.docx` - Word version

These files are also auto-generated by GitHub Actions on every push to main.

## Workflow

1. Edit `cv.qmd` with your updates
2. Test locally: `quarto render`
3. Commit and push to GitHub
4. GitHub Actions automatically renders and deploys
5. View at: https://yourusername.github.io/cv/
